# Definición de Listo (DoR) - Checklist Detallado

Utiliza este checklist para profundizar en la auditoría de cada historia.

## Criterios de Aceptación
- [ ] ¿Están escritos en formato "Dado que... Cuando... Entonces..."? (Opcional pero recomendado)
- [ ] ¿Son atómicos (se puede probar cada uno por separado)?
- [ ] ¿Cubren casos de error o flujos alternos?

## Diseño y UX
- [ ] Si aplica, ¿hay links a Figma/mockups actualizados?
- [ ] ¿Se definen comportamientos de UI específicos?

## Dependencias Técnicas
- [ ] ¿Se han identificado los endpoints de API necesarios?
- [ ] ¿Requiere cambios en la base de datos o infraestructura?
- [ ] ¿Están resueltos los bloqueos con otros equipos?

## Documentación
- [ ] ¿La descripción es clara y libre de ambigüedades?
- [ ] ¿Se menciona el impacto en otras funcionalidades existentes?
